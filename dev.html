Development and Compliance

Planning, testing, and website responsibility notes

This page explains how the website was designed, developed, tested, and checked against the assignment issues.

Development process

The website was planned first by listing the required pages: Home, About Us, Available Pets, Adoption Information, Gallery, Contact Us, Thank You, and Development and Compliance. After that, a consistent navigation menu and footer were designed so visitors can move between pages easily.

The design uses warm colours, large pet photos, simple card layouts, and readable spacing. The site was developed as static HTML, CSS, and JavaScript. The contact form uses basic browser validation and sends the user to the Thank You page after submission.

Testing summary

  • Page links: checked that all navigation and footer links open the correct pages.
  • Navigation: checked desktop menu and mobile menu button behaviour.
  • Images: checked all pet and gallery images load and include alt text.
  • Form submission: checked the Contact Us form requires name, email, and comments, then goes to thankyou.html.

Privacy, security, copyright, and censorship

Privacy: The contact page explains why user details are collected. Only name, email, and comments are asked for, so unnecessary personal information is avoided.

Security: The form uses required fields and email input type. A real website would also need server-side validation, spam protection, and secure storage if messages were saved.

Copyright: Pet images were downloaded from Unsplash into the local images folder. The HTML includes source comments near the image tags, and the footer acknowledges the source. Images are used for an educational static website.

Content appropriateness: Animal rescue content can be upsetting if it shows neglect or injury. This site uses positive and safe images suitable for general visitors and younger audiences.
